Anypoint Design Center (Design Center) enables you to create apps visually. To use Design Center,
work with your Anypoint Platform administrator to ensure that you have
a xref:access-management::environments.adoc#to-create-a-new-environment[design
environment].
For more information, see the xref:design-center::fd-tour.adoc[Flow Designer
Tour].
To create an app in Design Center:

* Configure the input source (trigger) for your app.  
* Add the connector as a component to process the input for the app.

For information about Design Center field values, see the
the xref:salesforce/salesforce-b2c-commerce-cloud-connector-reference.adoc[Connector Reference].

== Configure the Input Source Trigger

. In Design Center, click *Create*.
. Click *Create new application*.
. Specify a value for *Project name*, and click *Create*.
. Click *Go straight to canvas* to exit *Let's get started*.
. Click the trigger card.
. Use HTTP Listener as the trigger.
. Complete these required fields:
+
[%header%autowidth.spread]
|===
|Field |Description
|*Protocol*|Protocol selected for the HTTP connector: either HTTP or HTTPS.
|*Host*|IP address where your Mule application listens for requests.
|*Port*|Port address where your Mule application listens for requests.
|*Base Path*|Path where your Mule application listens for requests.
|===
+
. Click *Test*.

Design Center automatically saves all changes.

== Configure the Target Component

. Click *+* next to the trigger card.
. In *Select a component*, search for Salesforce B2C Commerce Cloud Connector.
. Select the connector name.
. Select an operation for the connector and add the configurations.

* Complete these required fields for the `Client_Id` Connection Provider:
+
[%header%autowidth.spread]
|===
|*Field* |Description 
|*Base URI*|URL to access API
|*Client Id*|Client ID for authentication
|===
+
* Complete these required fields for OAuth 2 Application Connection Provider:
+
[%header%autowidth.spread]
|===
|Field |Description  
|*Base URI*|URL to access API
|*Username*|Username of commerce cloud account
|*Password*|Password of commerce cloud account
|*Token URI*|URL to access token
|*Client Id*|Client ID of commerce cloud account
|*Client Secret*|Client secret of commerce cloud account
|===
+
* Complete these required fields for Customer Auth Connection Provider:
+
[%header%autowidth.spread]
|===
|Field |Description  
|*Base URI*|URL to access API
|*Username*|Username of commerce cloud account
|*Password*|Password of commerce cloud account
|*Client Id*|Client ID of commerce cloud account
|*Type*|Customer Type (Guest or Credentials)
|===
+
. Click *Test*.
